## 🎯 Tujuan Sesi

Implementasi File Upload + RAG Knowledge Base untuk AI Agent Engine.

---

## ✅ Yang Telah Dilakukan

1. **Text Extraction Pipeline**:
   - PDF: `pdf-parse@1.1.1` — kompatibel dengan Node.js 26
   - DOCX: `mammoth` — ekstrak teks dari dokumen Word
   - TXT: read langsung sebagai UTF-8
   - Type declarations di `types/pdf-parse.d.ts`

2. **Embedding & Vector Search**:
   - OpenAI `text-embedding-3-small` untuk generate embeddings
   - Cosine similarity untuk mencari dokumen relevan
   - Threshold similarity 0.3, top-3 dokumen teratas
   - Embedding disimpan sebagai JSON string di SQLite

3. **API Endpoints**:
   - `GET /api/documents` — List dokumen user
   - `POST /api/documents` — Upload + extract text + generate embedding + save
   - `DELETE /api/documents/[id]` — Hapus dokumen
   - Validasi: file size max 10MB, tipe PDF/DOCX/TXT saja

4. **RAG Integration di Chat**:
   - `/api/chat` sekarang otomatis mencari dokumen relevan
   - Query embedding dari pesan user → cosine similarity → inject konteks ke system prompt
   - Non-fatal: jika RAG gagal, chat tetap berjalan normal

5. **Knowledge Base UI (DocumentsPanel)**:
   - Panel slide dari kanan (toggle book icon di top bar)
   - Upload file via drag area (PDF, DOCX, TXT)
   - Daftar dokumen dengan ikon tipe, ukuran, tanggal
   - Hapus dokumen dengan tombol hover
   - Loading state saat upload & processing

## 🧠 Keputusan & Catatan Penting

1. **pdf-parse@1.1.1** digunakan karena pdf-parse v2 membutuhkan DOMMatrix (browser API) yang tidak ada di Node.js 26
2. **Embedding di SQLite** disimpan sebagai JSON string — untuk dev sudah cukup. Untuk production perlu pgvector di PostgreSQL
3. **RAG non-blocking**: Error di RAG tidak mengganggu chat utama (try-catch)
4. **Threshold similarity 0.3** — cukup rendah agar dokumen semi-relevan tetap terdeteksi
