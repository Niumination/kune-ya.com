## 🎯 Tujuan Sesi

Setup database ORM, sistem autentikasi, middleware, dan halaman profil yang terhubung ke database.

---

## ✅ Yang Telah Dilakukan

1. **Prisma ORM (SQLite dev)**:
   - Install Prisma 6 + @prisma/client 6 (stabil)
   - Schema: User, Account, Session, VerificationToken (kompatibel NextAuth.js + PostgreSQL)
   - Migration `init` sukses, database `prisma/dev.db` siap
   - `lib/db.ts` — Prisma client singleton

2. **NextAuth.js v5 (Auth.js)**:
   - Install next-auth@beta (v5.0.0-beta.31)
   - Credentials provider (email + password)
   - JWT strategy untuk session
   - bcryptjs untuk hashing password
   - `lib/auth.ts` — konfigurasi NextAuth + callbacks

3. **Auth Pages**:
   - `app/api/auth/[...nextauth]/route.ts` — NextAuth API route
   - `app/api/auth/register/route.ts` — Register API dengan validasi (email/username unik, format username)
   - `app/(auth)/login/page.tsx` — Login form client-side dengan error handling
   - `app/(auth)/register/page.tsx` — Register form client-side dengan validasi

4. **Middleware**:
   - `middleware.ts` — Proteksi route `/dashboard`, public routes dibiarkan akses bebas
   - Redirect ke `/login` dengan callbackUrl jika belum login

5. **SessionProvider**:
   - `components/layout/AuthProvider.tsx` — Client wrapper SessionProvider
   - Terintegrasi di root layout

6. **AuthButton**:
   - `components/layout/AuthButton.tsx` — Menampilkan login/register atau profil/logout berdasarkan session
   - Terintegrasi di navbar landing page

7. **Halaman Profil `[username]`**:
   - Query database dengan Prisma
   - Menampilkan nama, username, role, bio, tanggal bergabung
   - Deteksi pemilik profile (isOwner)
   - CTA berbeda untuk pemilik vs pengunjung

## 🧠 Keputusan & Catatan Penting

1. **Prisma 6** dipilih karena Prisma 7 (rilis 2026) memiliki breaking changes besar (prisma.config.ts, adapter-based) yang kurang stabil
2. **SQLite untuk dev** — zero setup, file-based, mudah di-switch ke PostgreSQL untuk production
3. **NextAuth.js v5 beta** — kompatibel dengan Next.js 15 App Router dan memiliki tipe yang baik
4. **Credentials only** — untuk MVP, cukup email+password. OAuth (Google) bisa ditambah nanti
5. **Middleware selektif** — hanya `/dashboard` yang diproteksi, landing page dan profile publik tetap bisa diakses siapa saja
